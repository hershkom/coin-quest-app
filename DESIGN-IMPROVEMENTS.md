## מצב קיים (לקרוא לפני שנוגעים)
- **קבצים:** כל העיצוב ב-`styles.css` (~490 שורות, בלי build step). מחלקות עיקריות: `.hero`, `.card`, `.bigbtn`, `.btn`, `.chore-row`, `.bottomnav`, `.modal`, `.streak-banner`, `.kid-card`.

## V1 — כיבוי תנועה מלא: `prefers-reduced-motion` + השלמת calm-mode 🌿 (S)
**מטרה:** מי שהגדיר במכשיר "הפחת תנועה" יקבל אפס אנימציות דקורטיביות אוטומטית — בלי קשר למצב רגוע.
1. ב-`styles.css` להוסיף בסוף:
```css
@media (prefers-reduced-motion: reduce){
  *,*::before,*::after{animation-duration:.01ms !important;animation-iteration-count:1 !important;transition-duration:.01ms !important;}
}
```
2. להרחיב את רשימת `.app.calm-mode` הקיימת כך שתכסה גם אנימציות שנוספו מאוחר: `.learn-dot.active` (transform), `@keyframes bob` על `.big-coin`/`.picker-logo` (כבר כלול), `.scan-line` (פונקציונלי — להשאיר), `#gameTimerChip.warning` (פונקציונלי-בטיחותי — להשאיר).
3. `coinBurst()` ב-app.js: כשמתקיים `matchMedia('(prefers-reduced-motion: reduce)').matches` — לא לירות חלקיקים בכלל (היום רק calmMode מפחית).
**קריטריון קבלה:** הפעלת reduce-motion ב-OS ⇒ המטבע בהירו לא קופץ, פתיחת view בלי אנימציית pop, סימון מטלה עדיין עובד ומציג טוסט.

## V2 — שדרוג ערכת המיינקראפט לבלוקים אמיתיים 🧱 (M)
**מטרה:** היום ה"מיינקראפט" של אריאל הוא בעיקר פונט פיקסלים + פסים עדינים. להפוך אותו למשכנע באמת בעזרת שפת הצורות של המשחק — בלי לגעת בערכת ברירת-המחדל של שאר הילדים.
1. **פינות מרובעות:** בתוך `.app.minecraft-mode` לדרוס `border-radius` ל-4px (לא 0 — נגיעה רכה) על `.card,.btn,.bigbtn,.chore-row,.modal,.kid-card,.game-row`.
2. **מסגרת bevel בסגנון כפתור מיינקראפט:** על `.app.minecraft-mode .btn` ו-`.bigbtn`:
```css
border:3px solid; border-color:rgba(255,255,255,.45) rgba(0,0,0,.35) rgba(0,0,0,.35) rgba(255,255,255,.45);
```
(בהיר למעלה/שמאל, כהה למטה/ימין — זה כל הסוד של כפתור מיינקראפט). ב-`:active` להפוך את הכיוונים.
3. **פס אדמה+דשא בראש ההירו:** ל-`.app.minecraft-mode .hero` להוסיף `border-top:8px solid;` בשכבות: `border-image:linear-gradient(180deg,#52C64B 0 40%,#8B7355 40% 100%) 1;` או פשוט שני `box-shadow: inset 0 4px 0 #52C64B, inset 0 8px 0 #8B7355`.
4. **הערה קריטית — הפונט:** ל-Press Start 2P **אין גליפים בעברית** — טקסט עברי נופל בשקט לפונט ברירת המחדל וזה בסדר (המספרים והאנגלית מקבלים פיקסלים). אסור "לתקן" את זה ע"י פונט אחר בלי בדיקה; אם רוצים כותרות עבריות פיקסליות — לנסות את Google Fonts **"Rubik Pixels"** (יש תמיכה בעברית) *רק* על `.section-title` במצב מיינקראפט, ולבדוק קריאוּת עם המשתמש לפני שמרחיבים.
**קריטריון קבלה:** בפרופיל אריאל הכפתורים נראים כמו כפתורי המשחק (בהיר-כהה), לנועה שום שינוי; צילום מסך לפני/אחרי.

## V3 — רקע חי: שמיים, עננים ועומק במקום אימוג'י סטטי ☁️ (M)
**מטרה:** `#mc-deco` היום = 3 שורות אימוג'י שקופים. להחליף ברקע שנותן תחושת עולם.
1. להחליף את תוכן `addMinecraftDecorations()` (app.js ~562) ב-2-3 עננים פיקסליים מ-SVG inline (מלבנים לבנים מדורגים, `shape-rendering:crispEdges`) שנעים לרוחב לאט מאוד (`animation: cloud-drift 90s linear infinite` — איטי=לא מסיח), ושורת בלוקים דקה (דשא/אדמה, גרדיאנטים חוזרים) בתחתית המסך מאחורי התוכן.
2. כיבוי מלא של תנועת העננים ב-calm-mode וב-reduced-motion (`.app.calm-mode ~ #mc-deco *` לא עובד כי mc-deco מחוץ ל-.app — לכן להוסיף למחלקת ה-body: כש-calm פעיל app.js כבר מוסיף class ל-.app; להוסיף גם `document.body.classList.toggle('calm')` באותו מקום, ולכבות עם `body.calm #mc-deco *{animation:none}`).
3. גרסת לילה: אחרי `state.anchored.sleep_time` (שעת שינה, קיים ב-state) — שמיים כהים + ירח + כוכבים סטטיים במקום עננים. בדיקת השעה כבר קיימת בלוגיקת ה-day-strip (`renderDayStrip`) — להשתמש באותו חישוב, לא לכתוב חדש.
**קריטריון קבלה:** רקע חי בפרופיל אריאל ביום, לילי אחרי שעת השינה, קפוא לגמרי במצב רגוע.

## V4 — מטבע שעף אל הארנק (מיקרו-אינטראקציה מרכזית) 🪙 (M)
**מטרה:** הרגע הכי חשוב באפליקציה — הרווחת מטבע — יקבל משוב פיזי מדויק במקום פיצוץ חלקיקים כללי.
1. פונקציה חדשה `coinFly(fromEl)` ב-app.js: יוצרת `<div>` עגול בסגנון המטבע (radial-gradient זהב קיים ב-`.balance-pill .coin`), ממוקם `position:fixed` על קואורדינטות `fromEl.getBoundingClientRect()`, מונפש ב-Web Animations API (`el.animate(...)`) אל מיקום `#balTop` (הפיל בטופבר), scale קטן + fade, ~700ms, ואז `remove()` ועדכון המספר.
2. לחבר: `markChore` (הכפתור שנלחץ), `mathCheck` (תיבת התשובה), `answerLearningQuestion` (כפתור התשובה — הפרמטר `btnEl` כבר קיים!). כשאין אלמנט מקור (QR, שער) — ליפול ל-`coinBurst()` הקיים.
3. במצב רגוע/reduced-motion: בלי טיסה — רק עדכון המספר עם הבהוב עדין (`transition:color`).
**קריטריון קבלה:** סימון מטלה ⇒ מטבע אחד עף מהשורה אל הפיל למעלה והמאזן מתעדכן בדיוק כשהוא "נוחת".

1. ל-`#view-learn` רקע כרטיס כהה-חום עדין (גרדיאנט `#3E2F23→#5a4632`, טקסט בהיר) רק סביב אזור השאלה, עם 2-3 "עורקי יהלום" — ריבועים טורקיז קטנים ב-`box-shadow` פינתיים.

**קריטריון קבלה:** ילד עם 30 תשובות נכונות רואה מכוש אבן ופס התקדמות לברזל; הבדיקות הקיימות של הלמידה עדיין עוברות.
